Reports SQL code style issues detected by SQLFluff linter.

This inspection controls the external annotator that runs sqlfluff lint on your SQL files. When enabled, SQLFluff analyzes your code in real-time and highlights violations directly in the editor. Disabling this inspection turns off the annotator, stopping SQLFluff from running and hiding all reported errors.

Below are some common examples of errors reported by SQLFluff: 


-- L010: Keywords must be consistently upper case.
select id, name from users;
-- Fix:
SELECT id, name FROM users;


-- L001: Unnecessary trailing whitespace.
SELECT id, name FROM users;   
-- Fix: remove trailing spaces at end of line.


-- L003: Expected 4 spaces indentation, found 2.
SELECT
  id,
  name
FROM users;
-- Fix:
SELECT
    id,
    name
FROM users;


-- L009: Files must end with a single trailing newline.
SELECT id FROM users;
-- Fix: ensure file ends with exactly one newline character.


-- L014: Unquoted identifiers must be consistently lower case.
SELECT Id, Name FROM Users;
-- Fix:
SELECT id, name FROM users;

Configure the SQLFluff executable path and linter rules in SQLFluff Settings.
